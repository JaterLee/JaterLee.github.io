# Add History Note Skill

帮助用户新增五代十国史（Five Dynasties History）笔记：接收简短的文字 → 润色丰富 → 生成规范的 `.md` 文件 → 构建 JSON 数据。

## 触发条件

用户说"新增笔记"、"添加文稿"、"写一篇读史笔记"、"记录一段"、"add history note"，或者直接分享一段历史相关的文字/感悟时使用。

## 笔记类型

| 类型 | 标识 | 说明 | 必填字段 |
|------|------|------|----------|
| 📖 摘录 | `passage` | 历史书中的段落、史料引用 | title, date, source |
| 💭 感悟 | `reflection` | 个人读史感受、分析、思考 | title, date |

## 执行流程

### 1. 收集信息

如果用户没有一次性提供完整信息，按优先级询问：

1. **标题** — 简短有力，建议 8-20 字
2. **类型** — 根据内容自动判断（有出处/史料引用 → passage，个人观点/感悟 → reflection）
3. **来源**（仅 passage）— 如《旧五代史》《资治通鉴》等
4. **日期** — 默认今天
5. **标签** — 2-5 个关键词，如 后梁、朱温、读后感、思考

如果信息齐全，直接进入下一步。

### 2. 润色文字

这是核心步骤。用户写的文字可能很简略，需要：

- **扩充内容**：将要点展开成完整的段落。一句"朱温杀了皇帝"可以展开为"天祐元年（904年），朱温逼迫唐昭宗迁都洛阳，随后弑君，立年仅十三岁的李柷为唐哀帝。"
- **添加结构**：用空行分段，用 `>` 标记关键引用，用 `-` 或 `1.` 组织列表
- **使用 markdown**：重点用 `**加粗**`，书名用书名号，引用用 blockquote
- **保持史实准确**：不要编造历史细节。如果不确定具体年份或史实，保留用户原文不做扩充
- **语气匹配**：passage 保持客观叙述风格，reflection 可以用第一人称

**润色原则**：
- 保留用户的核心观点和表达风格
- 只补充使文章更流畅、更完整的内容
- 不要过度修饰，保持自然
- 中文使用全角标点（，。、；：？！）

### 3. 生成 frontmatter

根据内容生成 YAML frontmatter：

```yaml
---
id: <kebab-case-slug>       # 从标题自动生成，如 "李存勖灭梁" → "li-cunxu-mie-liang"
title: <标题>
type: <passage|reflection>
date: <YYYY-MM-DD>           # 默认今天
source: <出处>               # 仅 passage，如 "《旧五代史》卷二十八"
tags: [标签1, 标签2, 标签3]   # 2-5 个关键词
---
```

**id 生成规则**：
- 标题转拼音或英文关键词，小写，用 `-` 连接
- 中文标题取主要人物+事件，如"朱温篡唐" → `zhuwen-cuan-tang`
- 感悟类加 `reflection-` 前缀，如 `reflection-five-dynasties-cycle`

**tags 建议**：
- 涉及人物：标注人名
- 涉及朝代/政权：标注政权名
- passage 类型：标注史书类别
- reflection 类型：标注"读后感"或"思考"

### 4. 写入文件并构建

1. 将生成的内容写入 `content/history/<id>.md`
2. 运行 `node scripts/sync-history.js` 构建 JSON
3. 报告结果：
   - 文件路径
   - 笔记类型和标题
   - 构建结果（笔记总数）

## 示例

### 示例 1：用户提供简略摘录

**用户输入**：
> 新增笔记：朱温白马驿杀朝臣三十多人，李振说清流该变浊流

**润色输出** (写入 `content/history/baima-calamity.md`)：

```markdown
---
id: baima-calamity
title: 白马之祸
type: passage
date: 2026-07-19
source: 《旧唐书》卷二十下
tags: [朱温, 白马之祸, 唐朝, 士族]
---

天祐二年（905年），朱温在亲信李振的鼓动下，于白马驿（今河南滑县）将朝臣三十余人处死，投尸于黄河。

李振曾多次参加科举不第，对朝中公卿心怀怨恨。他向朱温进言：

> **"此辈常自谓清流，宜投入黄河，使为浊流。"**

朱温采纳了这一建议。被杀的三十余人多为唐朝忠臣与士族精英。

白马之祸标志着唐朝士族势力的彻底覆灭，为朱温篡唐扫清了最后的障碍。
```

### 示例 2：用户提供简短感悟

**用户输入**：
> 写个感悟：五代都很短命，因为武将太强了，皇帝镇不住

**润色输出** (写入 `content/history/reflection-warlord-era.md`)：

```markdown
---
id: reflection-warlord-era
title: 武夫当国——五代短命的根源
type: reflection
date: 2026-07-19
tags: [五代, 军阀, 思考]
---

五代（907年—960年）五十三年间更替了五个朝代、十四个皇帝，平均每个朝代仅存续十年左右。

观察这些朝代的兴衰，有一个惊人的规律：

1. **开国君主雄才大略**——能征善战的枭雄打下江山
2. **继承人无能或年幼**——第二任皇帝几乎守不住基业
3. **军阀叛乱夺位**——手握兵权的将领趁虚而入
4. **王朝迅速倾覆**——从强盛到灭亡往往只需几年

这个循环的根源在于权力的逻辑：**武夫当国的时代，忠诚只能靠武力维系。**

皇帝活着才能镇住军队，一旦去世或软弱，手下将领立刻取而代之。文官系统和正统观念在这个时代完全坍塌了——枪杆子里出政权，也出叛军。
```

## 注意事项

- `.md` 文件保存到 `content/history/` 目录
- 构建脚本路径：`node scripts/sync-history.js`（在仓库根目录运行）
- 生成 JSON 后，前端瀑布流会自动显示新卡片
- 如需在网站看到效果，还需要 `git commit && git push`
- 如果用户想修改已有笔记，直接编辑对应的 `.md` 文件再重新构建即可
